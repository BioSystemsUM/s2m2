Packaging¶

When you write several commands in a file to be executed as a program, this file is called a script. When a program is imported, it is called a module.

As a developer starts implementing several modules, they tend to organize them into what is called a package. So, basically, a package is a group of similar modules that usually have a similar purpose.

Fortunatelly, one can use packages already implemented. Commonly, these packages are available in different repositories (https://pypi.org/, https://anaconda.org/anaconda/repo). To install them, we can use pip or conda, the latter only if anaconda or miniconda is installed (https://docs.conda.io/projects/conda/en/latest/user-guide/install/index.html).
